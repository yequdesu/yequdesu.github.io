第一次博客作业

一、自我介绍

学号：052301319
我是 YeQuDesu，一个爱好睡觉但又总想着折腾点什么的人。
我在福大食堂最喜欢的一道菜是——玫瑰一层老顺德干蒸菜的干蒸猪五花（友情提示：千万别点鸡肉，全是骨头没肉！）。
目前循环播放最多的一首歌是《アンノウン・マザーグース》，旋律和歌词都让我很有共鸣。
一句想说的话：我太想进步了。

二、思考

2.1 初入大学时对计算机专业的畅想

2.1.1 我如何选择计算机专业

我其实没有特别明确的兴趣方向，只是觉得计算机专业很有创造力、有无限的发挥空间。喜欢自由的我，希望能在这个专业中找到表达自我、实现想法的方式，于是我来了。

2.1.2 课程与期待是否一致

说实话，一开始并不太一致。我原本期望大学课程更像项目驱动、动手实践的那种类型，但刚入学时却被一堆理论课和晚自习浇灭了热情，感觉仿佛又回到了高四。直到大二、大三，我才逐渐醒悟，开始自己做项目、探索实战，也决定考研来获取更多学习时间。这一阶段的课程也更贴近实践，这让我重新燃起了兴趣。

2.1.3 计算机是我喜欢/擅长的领域吗

我认为计算机确实是我喜欢且擅长的领域。我对逻辑推理和自学新知识有天然的兴趣，喜欢钻研问题直到彻底理解。比如在自制操作系统的过程中，我不断查阅资料、尝试实现，从内核到驱动都一点点摸索出来。这种“能从零创造”的感觉让我非常满足。虽然我不是天才型选手，但我相信持续的思考与练习能让我不断靠近真正的专业水平。

2.1.4 将来是否选择与计算机相关工作（含城市/公司/岗位）

我希望未来能进入硬件开发领域，成为一名硬件工程师。我希望用自己的技术为国家在“卡脖子”领域做出一点贡献。理想的路径是先在外地读研深造，再回到福建工作。薪资高低对我而言并不是最重要的，我更希望从工作中获得成长、体验和自我实现的价值。我想成为一个能独立支撑自己的力量，而不是仅仅依附于外部榜样的人。

2.2 与前人经验对照（已具备 / 还差的技能）

2.2.1 我已经具备的专业知识与技能

  • 熟悉 C / C++ / Python 等语言
  • 掌握 数据结构、操作系统、网络协议全栈 等核心基础
  • 具备一定的 硬件编程能力，了解 Verilog
  • 了解 makefile 构建系统 与 Git 团队协作
  • 有 自制操作系统 的实践经验
  • 接触过 PyTorch 与机器学习基础知识
  • 具备较好的 团队协作与独立解决问题能力

2.2.2 离合格本科生还差哪些

  • 算法与数学基础仍需系统强化
  • 工程实践经验不足，项目管理能力有待提升
  • 英语读写能力仍需提高，尤其是科研论文与技术文档阅读
  • 缺乏足够的科研经历与实战项目打磨
  • 编码风格与架构设计经验仍需积累
  • 需要更稳定的作息与执行力来支撑长期目标

2.3 大三选择（考研/工作/考公/出国）

2.3.1 我的选择

我选择 考研。
我想要更多的时间去积累和沉淀技术，而不是刚毕业就被推入职场成为“半吊子”。我希望真正具备解决复杂问题的能力，而不是被轻易取代。

2.3.2 相比同学的优势与劣势

优势：

  1. 内驱力强，自学能力突出，愿意花时间钻研底层原理。
  2. 心理素质好，不容易被困难击垮。
  3. 掌握的专业范围较广，具备软硬兼修的技术视野。

劣势：

  1. 绩点不高，缺乏早期规划。
  2. 不太擅长主动社交，常常独自学习。
  3. 有时会被完美主义与拖延所困扰。

2.3.3 针对选择的大三规划安排

  1. 每周保持 20小时 的考研复习时间，系统复习数学与专业课。
  2. 每周投入 5小时 算法练习（LeetCode 或 OJ 平台），稳步提升逻辑与代码能力。
  3. 每月至少完成 1个小型项目/技术实验，确保持续动手实践。
  4. 保持 每天早睡早起，避免熬夜拖垮状态。
  5. 关注研究生院校信息、导师方向，提前确定目标方向与初步科研计划。
  6. 适当参与社团或竞赛，提升表达与协作能力。

2.4 对近期热点或中国经济的看法

我认为中国经济正处于转型期，从高速增长转向高质量发展阶段。虽然短期内压力不小，但长期来看，科技创新仍是最大的底气。面对全球经济的不确定性，我们更需要提升自身的科研能力与核心竞争力。
对于一些国际热点事件（例如核废水排放、国际技术封锁等），我更感受到“技术自立”的重要性。只有在关键领域真正做到自主可控，国家的发展才不会被他人牵制。
我希望能以个人的努力，哪怕微小，也能参与到这种“技术独立”的浪潮中。

三、未来的期望

(3.1) 是否愿意为感兴趣/实战项目熬夜以及如何看团队熬夜回忆

我对“熬夜”这件事的态度是——如果值得，就去做。
当项目让我感到热血和投入时，我可以连续几天通宵；但我更希望团队是因为热爱和责任而奋斗，而不是被赶进度的压力逼迫。
我认为“凌晨四点的回忆”并不是浪漫，而是证明大家曾真诚投入过某件事。那种并肩调Bug、讨论方案、在黎明前看到成果的瞬间，是所有努力的意义所在。

(3.2) 对这门课的期待与每周投入时间

我希望这门课能更贴近“实战”与“团队协作”，让我们真正体验软件开发的完整流程，而不是纸上谈兵。我也希望能在团队中学会分工、沟通与责任感。
我计划每周投入 6–10小时 在这门课上：

  • 2小时复习课堂内容并整理笔记；
  • 4小时以上进行项目实践或团队协作；
  • 视任务情况延伸学习相关技术。
    我希望课程结束时，不仅学会写代码，更能学会如何“做项目、带团队、解决问题”。

结语：
我不想成为天才，只想一步步变强。希望未来的自己能像现在写下这篇博客时一样，仍然热爱、仍然渴望进步。
